# v0.6.4 — auto-inject project + design context into every review

## TL;DR

Every `conclave review` and `conclave audit` run now auto-loads a small,
bounded slice of the repo's own docs and passes them to every agent
(and, for design, reference images too). Council members finally see
**what the repo is FOR** before they critique what the diff says.

No config required. Drop any of these four files and the next review
picks them up:

| File | Who sees it |
|---|---|
| `README.md` (first 500 chars) | every agent |
| `.conclave/project-context.md` (full) | every agent |
| `.conclave/design-context.md` (full) | DesignAgent only |
| `.conclave/design-reference/*.png` (≤ 4 × 500KB) | DesignAgent vision mode |

## The failure this fixes

**eventbadge PR #20** hit it concretely: the council called
`with: cli-version: latest` a "CI config error" because the agents
couldn't see the reusable-workflow `uses:` line just above the hunk
— only the diff itself was in their prompt. No project context, no
way to know `cli-version: latest` is a reusable-workflow input, not a
GitHub Actions `uses: foo@latest` pin.

More generally: without a README slice or a project-context file,
agents judge every diff against a generic code-review rubric, not
against **this repo's intent**. False positives on intentional patterns
waste budget and erode trust.

## What changed

### `ReviewContext` gains three optional fields

```ts
// @conclave-ai/core — packages/core/src/agent.ts

interface ReviewContext {
  // ... existing fields ...
  /** README head + .conclave/project-context.md combined. */
  projectContext?: string;
  /** .conclave/design-context.md — brand/tone/a11y/persona. */
  designContext?: string;
  /** .conclave/design-reference/*.png — brand "good" images. */
  designReferences?: Array<{ filename: string; bytes: Buffer | Uint8Array }>;
}
```

All three are optional. Absent-file silent-skip is the normal case.

### New loader module — `packages/cli/src/lib/project-context.ts`

```ts
loadProjectContext(cwd, { readmeMaxChars? })
  → { projectContext?: string }

loadDesignContext(cwd, { maxReferences?, maxImageBytes? })
  → { designContext?: string, designReferences?: [...] }
```

- Reads ONLY the 4 paths listed. No arbitrary repo scan, no remote
  fetch.
- Truncates README on a word boundary and appends `... (truncated)`.
- Silently drops oversize reference images and non-PNG files.
- ENOENT / permission errors never crash the review — loader returns
  an empty object.

### Both commands wire in the loader

- `packages/cli/src/commands/review.ts` — loads project context always;
  loads design context only when resolved domain is `design` or `mixed`.
- `packages/cli/src/commands/audit.ts` — same pattern, per-batch ctx
  carries the loaded fields so every batch's agents see them.

### All four agent prompts consume the fields

Each gets a new `# Project context\n{{projectContext}}\n\n...` block
ahead of the diff / audit payload. DesignAgent also gets
`# Design intent\n{{designContext}}` and — in Mode A (vision) —
renders `designReferences` as image blocks **before** the PR-specific
`visualArtifacts`, labeled "Brand reference".

Unchanged: the technical rubric portion of each system prompt.

### Config schema — optional `context` section

```json
{
  "version": 1,
  "context": {
    "readmeMaxChars": 500,
    "maxDesignReferences": 4,
    "maxDesignImageBytes": 512000,
    "includeDesignReferences": true
  }
}
```

Zod-validated, all keys optional with sensible defaults. Omitting the
section keeps current behavior. **No `.conclaverc.json` regeneration
required** — auto-inject is purely additive.

## Example: `.conclave/project-context.md`

```markdown
# eventbadge

**What it is:** Printable badge generator for small conferences (~300
attendees). Next.js frontend, Supabase backend, Vercel deploy.

**Intentional patterns:**

- `.github/workflows/review.yml` uses OUR OWN reusable workflow at
  `seunghunbae-3svs/conclave-ai/.github/workflows/review.yml@v0.4`.
  The `cli-version: latest` input is a reusable-workflow parameter,
  NOT a `uses: foo@latest` pin. Do not flag it.
- All user-facing strings go through `t()` for i18n. Hardcoded
  strings in API routes + error logs are fine.

**What NOT to block on:**

- Missing CHANGELOG entries on dep bumps.
- Snapshot-test churn from shared-primitive changes.

**Audience:** small-conference organizers, not developers.
```

## Example: `.conclave/design-context.md`

```markdown
# Brand + UX target

**Brand:** Calm, print-forward, utilitarian. Think "Herman Miller
catalog", not "SaaS landing page". No gradients, no animation beyond
100ms transitions, no illustrated characters.

**Typography:** Inter (UI), IBM Plex Mono (badge numbers). Body
text 15px min.

**Color:** Two brand colors (navy `#0B2545`, sand `#EFE8D8`) plus
semantic tokens. Flag hardcoded hex outside that set.

**A11y target:** WCAG AA. Focus rings always visible. Never rely on
color alone.

**Persona:** High-school teacher running registration on a borrowed
laptop. NOT a developer — UI copy avoids jargon.

**Recent direction:** Feb 2026 we moved from bottom-nav to top-nav.
Re-introducing bottom-nav is a regression, not a new idea.
```

## Backward compatibility

- Three `ReviewContext` fields are optional; absent values yield
  prompts identical to v0.6.0.
- Loader silently skips missing sources — existing repos without a
  `.conclave/` directory see no change.
- Config `context` section is optional; `.conclaverc.json` without it
  still validates.
- DesignAgent's Mode A (vision), Mode B (text-UI), Mode C (skip), and
  audit paths all degrade gracefully when `designReferences` is absent.

## Acceptance smoke test

```bash
mkdir -p .conclave
cat > .conclave/project-context.md <<'EOF'
Our review.yml calls OUR reusable workflow. cli-version: latest is NOT
a GH Actions uses-pin; don't flag it.
EOF

corepack pnpm build
node packages/cli/dist/bin/conclave.js review --pr 20 \
  --domain code 2>&1 | head -20
```

The rendered prompt now contains a `# Project context` section. On the
original eventbadge PR #20 regression test, agents no longer blocker
`cli-version: latest`.

## What's NOT in this release

- No `--verbose` flag for context-loaded-from-disk logging — deferred
  to v0.6.5+.
- No remote fetching of context (e.g., from a shared org-level repo).
  Local files only by design.
- No changes to `central-plane` or `integration-telegram`.
- No npm publish in this PR. Tag + publish are a separate step.
- No auto-generation of `.conclave/project-context.md` from git log.
  Users write their own intent file.
