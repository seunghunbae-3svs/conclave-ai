# v0.13.5 — Dump patch + target-file snippet on apply conflict

Released **2026-04-26**.

## TL;DR

Pre-fix the conflict reason was just `patch failed: <file>:<line>` with the patch tempfile already unlinked. Operators reading CI logs couldn't tell whether the patch hit a context mismatch (line endings), missing file, or real merge collision.

Now we capture the patch (head 1500 chars) + first 12 lines of the target file before the temp delete, so the actual content mismatch (line endings / encoding / drift) is visible from CI logs alone.

Live-needed for `eventbadge#28` rework run 24957853174.
