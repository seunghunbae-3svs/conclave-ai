# v0.13.3 — Per-blocker status when "no applicable patches" bails

Released **2026-04-26**.

## TL;DR

Pre-fix: when worker patches all dropped (worker-error / conflict / secret-block), autofix logged a single line "all blockers skipped/conflict" with zero breakdown. Operators couldn't see whether the worker errored, the patch context-mismatched, or secret-guard fired.

Now dumps `[status] category @ file: reason` for every fix that was attempted. Same shape as `--dry-run`'s existing breakdown.
