# v0.13.13 — Fuzzy dedupe across agents reporting the same bug at off-by-one lines

Released **2026-04-27**.

## TL;DR

Live RC: `eventbadge#29` cycle 3 verdict had Claude flagging a `console.log` at line 18 and OpenAI flagging the SAME `console.log` at line 17 (off-by-one in OpenAI's attribution). Pre-fix dedupe key was `file|line|message[:60]` — both passed through. Autofix produced TWO patches; the first applied (with `patch -p1 --fuzz=3` fallback, offset 3 lines, exit 0); the second hit a "patch already applied" conflict because the line was gone. `applyFailed=true` → whole-iteration rollback → loop stalled at cycle 1/3.

## Fix

New rule in `dedupeBlockersAcrossAgents` (in addition to the exact key): collapse iff
- same `file`, AND
- `|line_a - line_b| ≤ 1`, AND
- the messages share a notable code-shaped token (≥4 chars, excludes a curated stopword list).

Conservative — keeps both blockers when the messages share no notable token (false-positive cost > false-negative cost). First-seen agent wins.

11 unit tests cover the `eventbadge#29` case, the must-NOT-collapse counterexamples (line diff > 1, file mismatch, no shared token, stopwords-only overlap), and the existing exact-dedupe + nit-filter behaviour.

## Tooling

Also: `bump-changed-packages.mjs` now expands the bump set to include every workspace dependent of a changed package. Previously, when `core` changed, only core bumped — cli@0.13.12 (frozen with `"@conclave-ai/core": "0.11.12"`) wouldn't pick up the fix until cli's source incidentally changed. Now cli + every other consumer of core bumps automatically, so consumers get the fix on the next install.
