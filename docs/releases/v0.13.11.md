# v0.13.11 — `conclave doctor` Telegram-webhook check

Released **2026-04-27**.

## TL;DR

Adds a 5th diagnostic line to `conclave doctor` that verifies the Telegram bot's registered webhook URL matches what the central-plane Worker expects.

The doctor never sees the bot token: it hits a new `GET /admin/webhook-status` endpoint on the Worker (Bearer-`CONCLAVE_TOKEN` auth, same gate as every other authed route) which calls `getWebhookInfo` server-side and returns `{ url, expected, matches, outcome, ... }`.

Outcome → severity:
- `bound` (matches=true) → OK
- `dropped` / `wrong-url` → FAIL with re-bind / find-the-offender hint
- `no-bot-token` / `telegram-unreachable` / 401 / 404 → WARN

7 worker-route tests + 8 doctor-check tests. Note: requires `wrangler deploy apps/central-plane` for the new endpoint to be live; the doctor degrades to WARN until the Worker side is redeployed.
