# v0.13.6 — Dedupe blockers by (file, line, message-prefix), not category

Released **2026-04-26**.

## TL;DR

Pre-fix `dedupeBlockersAcrossAgents` keyed on `category|file|message[:80]`. When 3 agents reported the SAME observable bug (stray `console.log` on `AddressSearch.jsx:2`) under DIFFERENT category labels (regression / logging / code-quality — agents disagree on taxonomy for the same bug), the dedupe key diverged and all 3 passed through. autofix then generated 3 patches, all targeting the same line.

First patch applied successfully. Second patch hit `git apply --check` failure because the line it expected at position 2 (console.log) was already replaced with empty line by the first apply. autofix rolled back, loop stalled at cycle 1/3 forever.

Fix: drop `category` from the dedupe key. The bug is the bug; the label is taxonomy that agents disagree on. Use `file|line|message[:60]` instead — line disambiguates same-file repeats; 60-char prefix differentiates distinct bugs at same line without locking out near-paraphrases from different agents.

(v0.13.13 later added a fuzzy pass for the off-by-one-line variant of this same RC.)
