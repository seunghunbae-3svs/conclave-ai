# v0.13.2 — Autofix `--no-notify` + apply conflict diagnostics

Released **2026-04-26**.

## TL;DR

Two issues caught during `eventbadge#25` live E2E:

1. Autofix's spawned `conclave review --pr N --json` (the verdict-fetcher used when `--verdict` isn't passed) was sending a fresh `notifyReview` to Telegram — duplicating the upstream review's notification. User-visible: "rework 0/3" appearing 3-4 times in chat per cycle. Fix: new `--no-notify` flag that stubs the notifier list to `[]`; autofix's default spawnReview now passes it.
2. Apply-conflict logging was a one-liner with no context. Now dumps the rejecting fix's file + reason tail to stderr.

Both are caller-side fixes (no protocol or DB changes).
