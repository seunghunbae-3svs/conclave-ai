# v0.13.4 — Stage only patched files (not all working-tree changes)

Released **2026-04-26**.

## TL;DR

Live-caught on `eventbadge#25` commit `22a0b99`: the autofix commit included `frontend_old/package-lock.json` (3018 lines), which the next review correctly flagged as an environment/compatibility blocker — blocking loop closure.

Root cause: autofix used `git add -A` to stage everything in the working tree. The rework workflow's dependency-install step creates lockfiles in subdirs that weren't there before; those got swept into the autofix commit alongside the actual patch.

Fix: track exactly which files autofix patched (`appliedPaths` from the unified-diff applies + `appliedFiles` from special-handler results) and `git add --` only those paths. Defense-in-depth fallback uses `git add -u` (tracked-only) when filesToStage is empty.
